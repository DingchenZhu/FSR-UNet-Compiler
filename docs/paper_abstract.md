# 摘要

专用卷积神经网络（Convolutional Neural Network, CNN）硬件加速器在图像超分辨率等计算机视觉任务中展现出显著的性能与功耗优势，但如何将主流深度学习框架训练的模型自动映射为加速器可执行的指令序列，至今仍是制约其规模化部署的工程瓶颈。现有方案普遍依赖手工编写的代码生成器：以本文的目标实现为例，约3,800行Python脚本将FSRCNN和SD-UNet两款模型的所有分块参数、地址偏移与循环次数以硬编码常量的形式直接嵌入代码，任何模型结构或输入分辨率的调整都需要开发者在海量常量中逐一定位修改，可维护性差、扩展代价高，且难以支持可变形卷积等非标准算子到硬件专用单元的正确映射。

针对上述问题，本文设计并实现了一个基于TVM Relay IR的编译器前端，面向自研CNN硬件加速器，将PyTorch/ONNX格式的神经网络模型自动转换为硬件伪指令序列。编译器采用四阶段流水线架构：前端解析（Relay IR）、层描述提取（LayerDesc）、分块规划（TilingPlan）与指令发射（伪指令流），各阶段输出经规范化接口传递，所有硬件约束集中建模于各层之间，彻底消除了手写方案中散落各处的硬编码常量。在定制化算子支持方面，本文提出OffsetGenerator子图融合Pass，基于"池化→卷积（输出通道=18）→可变形卷积"的三层结构性模式匹配，将偏移量生成子网络识别并融合为专用的`offset_gen`算子，确保其DataStorer指令以正确的目标地址（`dest=offset_reg`）写出，从语义上打通了可变形卷积硬件加速路径的数据通路。针对SD-UNet中的分组卷积（Grouped Convolution），本文提出双级循环发射框架，支持groups=2/8四种循环结构模式；引入全高度流式调度（Full-height Streaming）模式，通过TVM形状推断自动传递AveragePool下采样带来的特征图尺寸变化，覆盖编解码器对称结构；实现DepthToSpace透明化注入与Pool-while-store机制，对应硬件DataStorer并行像素重排与池化写出功能。此外，本文归纳出覆盖全部层类型的五种分块模板（Template C/D/E/F及offset\_gen专用模板）、实现了ping-pong双缓冲的逐层正确交替分配，以及`acc_mode`/`store_mode`量化模式的自动推导；针对输出层的mask-store模式建立了`base_addr_pooling`与`base_addrs_res`双通道感知互换规则，引入`TilingPlan.ds_last_transfer_num`机制以支持分组卷积每组末尾DataStorer的`transfer_num=0`结束信号协议；并在Post-Pass中实现了基于生产者-消费者依赖分析的虚拟寄存器分配与多帧流水调度指令序列设计。

在两个目标网络上的完整验证表明，编译器实现了与黄金参考的功能性等价：（1）**FSRCNN超分辨率网络**（输入尺寸$1\times1\times36\times64$，12层，含4个可变形卷积路径）：生成1,273/1,274条伪指令（分别对应独立推理/流水线模式），与手写黄金参考（`sd_sr_codegen.py`的`sr_inst()`函数）在QuantLoader、DataLoader、WeightLoader、DataStorer、OffsetLoader、OffchipDataStorer六类指令的数量上全部零差值精确匹配，剩余字段差异经分类分析全部归属外部输入依赖（`bas_addr`、`quant_mode`）或ISA模板精化项；（2）**SD-UNet超分辨率网络**（USR\_Net\_109\_nopad.onnx，输入$1\times1\times144\times256$，19层Conv，含groups=2/8分组卷积、AveragePool×4、DepthToSpace×5、Concat跳跃连接×4）：生成17,155/17,155条指令，与黄金参考（`sd_inst()`函数）精确匹配，剩余14,664个字段差异经逐层multiset分析全部确认为非功能性（WeightLoader `is_new`字段顺序调度差异及QuantLoader寄存器编号调度差异，均不影响硬件计算结果）。编译器以约800行通用框架代码取代原有3,800行硬编码脚本，同时支持两个网络的完整指令生成，具备面向任意ONNX/PyTorch模型的扩展能力。

**关键词**：TVM；编译器前端；卷积神经网络；硬件加速器；可变形卷积；分组卷积；指令生成

# Abstract

Dedicated convolutional neural network (CNN) hardware accelerators offer substantial advantages in throughput and power efficiency for computer vision tasks such as image super-resolution, yet automatically mapping models trained in mainstream deep learning frameworks to accelerator-executable instruction sequences remains an engineering bottleneck that limits large-scale deployment. Existing solutions rely predominantly on hand-written code generators: the reference implementation targeted in this work comprises approximately 3,800 lines of Python code that hard-codes all tiling parameters, address offsets, and loop counts for two fixed model architectures (FSRCNN and UNet), making any change to model structure or input resolution a labor-intensive and error-prone process. Such hand-written generators are further incapable of correctly mapping non-standard operators such as deformable convolution to hardware-specific functional units.

This paper presents the design and implementation of a TVM Relay IR-based compiler frontend targeting a custom CNN hardware accelerator. The compiler translates PyTorch and ONNX models into hardware pseudo-instruction sequences through a four-stage pipeline: frontend parsing (Relay IR), layer description extraction (LayerDesc), tiling planning (TilingPlan), and instruction emission. All hardware constraints are centrally modeled at the interfaces between stages, eliminating the scattered hard-coded constants characteristic of hand-written generators. To support non-standard operators, an OffsetGenerator subgraph fusion pass is proposed, which identifies the three-layer structural pattern (average pooling → convolution with 18 output channels → deformable convolution) and fuses the offset generation sub-network into a dedicated `offset_gen` operator, ensuring its DataStorer instruction targets the hardware offset register (`dest=offset_reg`) and correctly activating the deformable convolution hardware path. Four tiling templates (Template C/D/E/F) are derived to cover all layer types in the target models. Correct per-layer ping-pong buffer alternation, automatic derivation of `acc_mode`/`store_mode` quantization fields, a mode-aware swap rule for `base_addr_pooling` and `base_addrs_res` in mask-store output layers, and a `TilingPlan.ds_last_transfer_num` mechanism to support the `transfer_num=0` group-end protocol in grouped convolutions are also presented, along with a post-pass implementing producer-consumer dependency analysis, virtual register allocation, and multi-frame pipeline scheduling.

Validation on two target networks confirms full functional equivalence with the hand-written golden references. (1) **FSRCNN** (input shape $1\times1\times36\times64$, 12 layers including 4 deformable convolution paths): the compiler generates 1,273/1,274 pseudo-instructions (standalone/pipeline modes), achieving zero-difference exact match with `sr_inst()` across all six instruction-type counts (QuantLoader, DataLoader, WeightLoader, DataStorer, OffsetLoader, OffchipDataStorer); remaining field-level differences are categorized as external-input dependencies (`bas_addr`, `quant_mode`) or ISA template refinement items. (2) **SD-UNet** (USR\_Net\_109\_nopad.onnx, input $1\times1\times144\times256$, 19 Conv layers with groups=2/8 grouped convolutions, AveragePool×4, DepthToSpace×5, and Concat skip connections×4): the compiler generates exactly 17,155/17,155 instructions matching `sd_inst()` precisely; the remaining 14,664 field differences are systematically confirmed as non-functional via per-layer multiset analysis (WeightLoader `is_new` scheduling artifacts and QuantLoader register index differences, neither of which affects hardware computation). The compiler replaces approximately 3,800 lines of hard-coded script with roughly 800 lines of general-purpose framework code, supporting full instruction generation for both target networks simultaneously. This work offers a reusable engineering reference for neural network compiler design targeting custom hardware accelerators.

**Keywords**: TVM; compiler frontend; convolutional neural network; hardware accelerator; deformable convolution; grouped convolution; instruction generation
