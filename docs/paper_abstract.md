# 摘要

约3,800行Python脚本——这是本文研究出发点的现实写照。目标硬件加速器的参考代码生成器`sd_sr_codegen.py`将FSRCNN和SD-UNet两款模型的全部分块参数、地址偏移与循环次数以硬编码常量的形式直接写死在代码中。一旦输入分辨率变动，开发者只能在海量数字中逐一定位修改；若要新增一款模型，几乎等同于从头重写整个脚本；而可变形卷积（Deformable Convolution）等非标准算子所需的硬件专用数据通路，更无法通过简单的逐算子翻译来覆盖。

本文针对上述问题，设计并实现了一个基于TVM Relay IR的编译器前端，面向自研CNN硬件加速器。编译器以四级流水线为骨架：前端解析（Relay IR）→ 层描述提取（LayerDesc）→ 分块规划（TilingPlan）→ 指令发射（伪指令流）。各级之间以规范化数据结构为接口，所有硬件约束集中建模于层次边界，从根本上消除了手写方案中散落各处的硬编码常量。前端提供ONNX（`load_onnx`）和PyTorch（`load_pytorch`）双入口，以Relay IR为统一中间表示，经`relay.transform.InferType()`形状推断后驱动后续各级。

在算子支持方面，本文的核心贡献是OffsetGenerator子图融合Pass。该Pass依据"池化→卷积（输出通道=18）→可变形卷积"三层结构性模式，将偏移量生成子网络识别并融合为专用`offset_gen`算子，确保其DataStorer指令以`dest_buffer_idx='offset_reg'`写入偏移寄存器，而非误入普通数据buffer。没有这一Pass，OffsetLoader在硬件上永远读不到有效偏移量——可变形卷积的加速路径形同虚设。针对SD-UNet，本文进一步引入全高度流式调度（Full-height Streaming）模式：`tile_h=None`分支通过TVM形状推断自动传递AveragePool下采样带来的特征图尺寸折半，覆盖编解码器对称结构，分流逻辑仅在`choose_tiling()`内的一处条件判断中体现。分组卷积（Grouped Convolution）方面，双级循环发射框架以统一的`_emit_group_conv`路径处理groups=2/8的四种循环结构，DepthToSpace与pool-while-store则分别以透明化字段注入（`is_pixelshuffle`、`is_pooling=1`）的方式实现，不增减指令条数。

在分块与调度层面，本文归纳出覆盖目标网络全部层类型的五种Tiling模板（Template C/D/E/F及`offset_gen`专用模板），并实现了ping-pong双缓冲的逐层正确交替分配、`acc_mode`/`store_mode`量化模式的自动推导规则，以及针对输出层mask-store模式的`base_addr_pooling`与`base_addrs_res`双通道互换规则。`TilingPlan.ds_last_transfer_num`机制专门处理分组卷积每组末尾DataStorer的`transfer_num=0`结束信号协议。Post-Pass基于生产者-消费者依赖分析完成虚拟寄存器分配，并生成多帧流水调度指令序列。

验证结果如下。**FSRCNN超分辨率网络**（输入$1\times1\times36\times64$，12层，含4个可变形卷积路径）：生成1,273/1,274条伪指令（独立推理/流水线两种模式），QuantLoader、DataLoader、WeightLoader、DataStorer、OffsetLoader、OffchipDataStorer六类指令数量与黄金参考`sr_inst()`逐项对齐，指令数对齐验证完成。**SD-UNet超分辨率网络**（`USR_Net_109_nopad.onnx`，输入$1\times1\times144\times256$，19层Conv，含groups=2/8分组卷积、AveragePool×4、DepthToSpace×5、Concat跳跃连接×4）：生成17,155/17,155条指令，与黄金参考`sd_inst()`精确匹配；剩余14,664个字段差异经逐层multiset分析全部确认为非功能性——约93%源于WeightLoader `is_new`字段的顺序调度差异，约0.4%源于QuantLoader寄存器槽位选择差异，均不影响硬件计算结果，功能性diff为0。编译器以约800行通用框架代码取代原有3,800行硬编码脚本，同时支持两个网络的完整指令生成。

**关键词**：TVM；编译器前端；卷积神经网络；硬件加速器；可变形卷积；分组卷积；指令生成

# Abstract

Dedicated convolutional neural network (CNN) hardware accelerators offer substantial advantages in throughput and power efficiency for computer vision tasks such as image super-resolution, yet automatically mapping models trained in mainstream deep learning frameworks to accelerator-executable instruction sequences remains an engineering bottleneck that limits large-scale deployment. Existing solutions rely predominantly on hand-written code generators: the reference implementation targeted in this work comprises approximately 3,800 lines of Python code that hard-codes all tiling parameters, address offsets, and loop counts for two fixed model architectures (FSRCNN and UNet), making any change to model structure or input resolution a labor-intensive and error-prone process. Such hand-written generators are further incapable of correctly mapping non-standard operators such as deformable convolution to hardware-specific functional units.

This paper presents the design and implementation of a TVM Relay IR-based compiler frontend targeting a custom CNN hardware accelerator. The compiler translates PyTorch and ONNX models into hardware pseudo-instruction sequences through a four-stage pipeline: frontend parsing (Relay IR), layer description extraction (LayerDesc), tiling planning (TilingPlan), and instruction emission. All hardware constraints are centrally modeled at the interfaces between stages, eliminating the scattered hard-coded constants characteristic of hand-written generators. To support non-standard operators, an OffsetGenerator subgraph fusion pass is proposed, which identifies the three-layer structural pattern (average pooling → convolution with 18 output channels → deformable convolution) and fuses the offset generation sub-network into a dedicated `offset_gen` operator, ensuring its DataStorer instruction targets the hardware offset register (`dest=offset_reg`) and correctly activating the deformable convolution hardware path. Four tiling templates (Template C/D/E/F) are derived to cover all layer types in the target models. Correct per-layer ping-pong buffer alternation, automatic derivation of `acc_mode`/`store_mode` quantization fields, a mode-aware swap rule for `base_addr_pooling` and `base_addrs_res` in mask-store output layers, and a `TilingPlan.ds_last_transfer_num` mechanism to support the `transfer_num=0` group-end protocol in grouped convolutions are also presented, along with a post-pass implementing producer-consumer dependency analysis, virtual register allocation, and multi-frame pipeline scheduling.

Validation on two target networks confirms full functional equivalence with the hand-written golden references. (1) **FSRCNN** (input shape $1\times1\times36\times64$, 12 layers including 4 deformable convolution paths): the compiler generates 1,273/1,274 pseudo-instructions (standalone/pipeline modes), matching `sr_inst()` exactly in the total instruction count and per-class counts of all six instruction types (QuantLoader, DataLoader, WeightLoader, DataStorer, OffsetLoader, OffchipDataStorer); instruction-count alignment has been verified. (2) **SD-UNet** (USR\_Net\_109\_nopad.onnx, input $1\times1\times144\times256$, 19 Conv layers with groups=2/8 grouped convolutions, AveragePool×4, DepthToSpace×5, and Concat skip connections×4): the compiler generates exactly 17,155/17,155 instructions matching `sd_inst()` precisely; the remaining 14,664 field differences are systematically confirmed as non-functional via per-layer multiset analysis (WeightLoader `is_new` scheduling artifacts and QuantLoader register index differences, neither of which affects hardware computation). The compiler replaces approximately 3,800 lines of hard-coded script with roughly 800 lines of general-purpose framework code, supporting full instruction generation for both target networks simultaneously. This work offers a reusable engineering reference for neural network compiler design targeting custom hardware accelerators.

**Keywords**: TVM; compiler frontend; convolutional neural network; hardware accelerator; deformable convolution; grouped convolution; instruction generation
