# 摘要

专用卷积神经网络（Convolutional Neural Network, CNN）硬件加速器在图像超分辨率等计算机视觉任务中展现出显著的性能与功耗优势，但如何将主流深度学习框架训练的模型自动映射为加速器可执行的指令序列，至今仍是制约其规模化部署的工程瓶颈。现有方案普遍依赖手工编写的代码生成器：以本文的目标实现为例，约3,800行Python脚本将FSRCNN和UNet两款模型的所有分块参数、地址偏移与循环次数以硬编码常量的形式直接嵌入代码，任何模型结构或输入分辨率的调整都需要开发者在海量常量中逐一定位修改，可维护性差、扩展代价高，且难以支持可变形卷积等非标准算子到硬件专用单元的正确映射。

针对上述问题，本文设计并实现了一个基于TVM Relay IR的编译器前端，面向自研CNN硬件加速器，将PyTorch/ONNX格式的神经网络模型自动转换为硬件伪指令序列。编译器采用四阶段流水线架构：前端解析（Relay IR）、层描述提取（LayerDesc）、分块规划（TilingPlan）与指令发射（伪指令流），各阶段输出经规范化接口传递，所有硬件约束集中建模于各层之间，彻底消除了手写方案中散落各处的硬编码常量。在定制化算子支持方面，本文提出OffsetGenerator子图融合Pass，基于"池化→卷积（输出通道=18）→可变形卷积"的三层结构性模式匹配，将偏移量生成子网络识别并融合为专用的`offset_gen`算子，确保其DataStorer指令以正确的目标地址（`dest=offset_reg`）写出，从语义上打通了可变形卷积硬件加速路径的数据通路。此外，本文归纳出覆盖全部层类型的四种分块模板（Template C/D/E/F）、实现了ping-pong双缓冲的逐层正确交替分配，以及`acc_mode`/`store_mode`量化模式的自动推导，并在Post-Pass中实现了基于生产者-消费者依赖分析的虚拟寄存器分配与多帧流水调度指令序列设计。

在FSRCNN超分辨率网络（输入尺寸$1\times1\times32\times64$）上的验证结果表明，编译器在独立推理模式下生成1,273条伪指令，在流水线模式下生成1,274条，与手写黄金参考（`sd_sr_codegen.py`的`sr_inst()`函数）在QuantLoader、DataLoader、WeightLoader、DataStorer、OffsetLoader、OffchipDataStorer六类指令的**数量**上全部实现零差值的精确匹配（指令类型数量级的匹配；字段级差异分类及外部依赖参数分析详见第六章）。编译器以约800行通用框架代码取代原有3,800行硬编码脚本，在保持指令正确性等价的同时，具备面向任意ONNX/PyTorch模型的扩展能力，将新模型的接入代价从数周的手工重写缩减为增量式的规则扩展。本工作为面向定制硬件加速器的神经网络编译器设计提供了一套可复用的工程参考框架。

**关键词**：TVM；编译器前端；卷积神经网络；硬件加速器；可变形卷积；指令生成

# Abstract

Dedicated convolutional neural network (CNN) hardware accelerators offer substantial advantages in throughput and power efficiency for computer vision tasks such as image super-resolution, yet automatically mapping models trained in mainstream deep learning frameworks to accelerator-executable instruction sequences remains an engineering bottleneck that limits large-scale deployment. Existing solutions rely predominantly on hand-written code generators: the reference implementation targeted in this work comprises approximately 3,800 lines of Python code that hard-codes all tiling parameters, address offsets, and loop counts for two fixed model architectures (FSRCNN and UNet), making any change to model structure or input resolution a labor-intensive and error-prone process. Such hand-written generators are further incapable of correctly mapping non-standard operators such as deformable convolution to hardware-specific functional units.

This paper presents the design and implementation of a TVM Relay IR-based compiler frontend targeting a custom CNN hardware accelerator. The compiler translates PyTorch and ONNX models into hardware pseudo-instruction sequences through a four-stage pipeline: frontend parsing (Relay IR), layer description extraction (LayerDesc), tiling planning (TilingPlan), and instruction emission. All hardware constraints are centrally modeled at the interfaces between stages, eliminating the scattered hard-coded constants characteristic of hand-written generators. To support non-standard operators, an OffsetGenerator subgraph fusion pass is proposed, which identifies the three-layer structural pattern (average pooling → convolution with 18 output channels → deformable convolution) and fuses the offset generation sub-network into a dedicated `offset_gen` operator, ensuring its DataStorer instruction targets the hardware offset register (`dest=offset_reg`) and correctly activating the deformable convolution hardware path. Four tiling templates (Template C/D/E/F) are derived to cover all layer types in the target models. Correct per-layer ping-pong buffer alternation, automatic derivation of `acc_mode`/`store_mode` quantization fields, and a post-pass implementing producer-consumer dependency analysis, virtual register allocation, and multi-frame pipeline scheduling are also presented.

Validation on the FSRCNN super-resolution network (input shape $1\times1\times32\times64$) demonstrates that the compiler generates 1,273 pseudo-instructions in standalone inference mode and 1,274 in pipeline mode, achieving exact zero-difference agreement with the hand-written golden reference (`sd_sr_codegen.py`) in the **count** of all six instruction classes: QuantLoader, DataLoader, WeightLoader, DataStorer, OffsetLoader, and OffchipDataStorer. This constitutes instruction-type-level count matching; field-level differences and their categorization (external-input dependencies vs. ISA template parameters awaiting refinement) are analyzed in Chapter 6. The compiler replaces approximately 3,800 lines of hard-coded script with roughly 800 lines of general-purpose framework code, maintaining equivalent instruction correctness while providing extensibility to arbitrary ONNX and PyTorch models. This work offers a reusable engineering reference for neural network compiler design targeting custom hardware accelerators.

**Keywords**: TVM; compiler frontend; convolutional neural network; hardware accelerator; deformable convolution; instruction generation
